## 🏛️ Décisions techniques

### TruthLevel comme enum numérique
- **Choix** : `enum TruthLevel { STRUCTURAL=0, DERIVED=1, SEMANTIC=2, HEURISTIC=3 }`
- **Alternatives écartées** : string union type, const object
- **Raison** : comparaison ordinale possible (`if (t >= TruthLevel.SEMANTIC)`), sérialisable en INTEGER SQLite, extensible sans breaking change

### Migration DB additive
- **Choix** : `ALTER TABLE ... ADD COLUMN` avec DEFAULT (v1→v2) plutôt que DROP/CREATE
- **Alternatives écartées** : recréer les tables, versionner le schéma entier
- **Raison** : backward compatible, données existantes préservées, migration auto au démarrage via `persistence/db.ts`

### RetrievalResult wrapper
- **Choix** : `{ chunk, score, trace }` wrapper autour de `SemanticChunk`
- **Alternatives écartées** : augmenter SemanticChunk directement, retourner un tuple
- **Raison** : sépare les concerns (chunk = donnée brute, trace = métadonnée retrieval), exploitable par telemetry future sans modifier les types core

### Session bias comme post-processing
- **Choix** : `applySessionBias()` appliqué après scoring, réordonne les résultats
- **Alternatives écartées** : modifier les embeddings, requêtes SQL biaisées
- **Raison** : aucun coupling fort avec le moteur de retrieval, désactivable, testable isolément
