## 🏛️ Décisions techniques

### Architecture entièrement in-memory / sans réseau réel
- **Choix** : Toute l'infrastructure distribuée est simulée en mémoire via `DistributedEventBus` avec ordering stable
- **Alternatives écartées** : WebSockets, gRPC local, worker threads avec channels
- **Raison** : Déterminisme garanti, replay reproductible, zéro dépendance externe, compatible avec l'architecture append-only existante

### LCG déterministe pour simulations réseau
- **Choix** : `NetworkPartitionSimulator` et `DeliveryScheduler` utilisent un LCG (Linear Congruential Generator) seedé
- **Alternatives écartées** : `Math.random()`, crypto random
- **Raison** : Permet de rejouer exactement les mêmes scénarios de chaos avec le même seed

### ConsensusEngine avec 4 modes
- **Choix** : `unanimous`, `majority`, `weighted`, `leader` — tous résolus de façon synchrone dès que tous les votes sont enregistrés
- **Alternatives écartées** : Raft/Paxos complets, timeout-based resolution
- **Raison** : Résolution déterministe sans timers, compatible avec le replay event-sourced existant

### Champs privés inutilisés → getters publics
- **Choix** : Ajouter `getExecutor()`, `getBus()`, `getConsensus()` pour satisfaire le TS language server
- **Alternatives écartées** : Supprimer les champs, préfixe `_` seul (insuffisant pour TS 6138)
- **Raison** : `noUnusedLocals: false` dans tsconfig mais le LS VS Code signale quand même les propriétés — les getters éliminent le warning et ajoutent une API utile

### Points d'attention
- Toute la simulation réseau est **synchrone et déterministe** — ne pas introduire de vrais `setTimeout` ou `Promise` dans les modules distribués
- `NetworkPartitionSimulator.partitionNode()` appelle `registry.isolateNode()` ET `bus.addPartition()` — les deux doivent rester synchronisés
- `ConsensusEngine.vote()` ne résout que lorsque **tous** les `voters` déclarés dans la proposal ont voté — si un node est isolé, utiliser `forceResolve()` ou `abort()`
- `SplitBrainDetector` s'abonne au bus dans son constructeur — instancier **après** le bus, pas avant
- Le seed LCG de `NetworkPartitionSimulator` est muté à chaque appel `nextRand()` — les tests avec seed fixe doivent respecter l'ordre des appels pour être reproductibles
